debounceTime operator
  • emits a notification from the source observable only after a particular time span has passed without another source emission
  • search --> user types the values, fast API calls
  • Delay /emit the next
search with RxJs debounceTime
take operator
  • Emits only the first count values emitted by the source observable
  • in simple terms, in debounceTime operator if we want restrict the count / limit the count we can use take operator
  • take will tell us how many values we can accept
takeWhile operator
  • it is similar to take operator, like an extension
  • takeWhile takes the value and checks the condition whether it is satisfying or not
  • emits values emitted by the source observable so long as each value satisies the given condition, and then completes as soon as this condition is not satisfied
  • take boolean vale from function in condition
takeLast operator
  • takes last specified values
  • waits for source to complete, then emits the last N values from the source, as specified by the count argument
first operator
  • It will give the first value emitted byt the source observable
  • it is useful when we are expecting fixed values like list categories, list of statuses
last operator
  • It will give the last value emitted byt the source observable
  • it is useful when we are expecting fixed values like list categories, list of statuses
elementAt operator
  • will give us single value from source observale at a specific index
  • array index starts with index 0
filter operator
  • filter operator will give the filtered values from the source observable based upon the predicate/condition function given
  • if we have an array with 10 elements, we want only 2 elements based upon a condition
  • filter((v)=>this.filterList(v))
distinct operator
  • it gives all the values from the source observable that are distinct when compared with the previous value
  • similar to set
  • simply provides distinct/unique values from the list (removes redundancy)
skip operator
  • skip operator will give back an observable that will skip the first occurance of count items taken as input
  • skips the specified no.of elements from list (from starting index)
  • if specified as 2, then it will skip first 2 elements from the list
count operator
  • it will give back an observable that will skip the first occurance of count items taken as input
  • count gives no.of elements available
max operator
  • max oparator will take an observable with all values and return an observable with maximum value
  • provides maximum number from list
  • list with numbers as elements
min operator
  • min oparator will take an observable with all values and return an observable with minimum value
  • provides minimum number from list
  • list with numbers as elements
